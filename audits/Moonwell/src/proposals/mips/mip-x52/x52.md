# MIP-X52: Upgrade xWELL Bridge Adapter

## Summary

This proposal upgrades the xWELL Bridge Adapter to integrate with Wormhole’s new
executor model, replacing the legacy relayer-based flow. The upgrade will
support both onchain and offchain quote paths to ensure compatibility across
environments, including Moonbeam, which will rely on offchain quotes.

## Background

The current xWELL bridge adapter relies on Wormhole’s legacy relayer design. As
Wormhole transitions to an executor-based architecture, remaining on the current
implementation introduces dependency on outdated infrastructure. Quote
mechanisms also vary by chain.

## Proposal

If approved, this proposal authorizes upgrading the xWELL Bridge Adapter to:

- Replace the legacy relayer / `processVAA` flow with the Wormhole executor
  model;
- Support both onchain and offchain quote paths;
- Use the offchain quote path for Moonbeam; and
- Maintain compatibility with existing xWELL crosschain bridging functionality.

This proposal also authorizes any necessary configuration and administrative
updates required to complete the upgrade.

## Rationale

This upgrade aligns Moonwell’s bridge infrastructure with Wormhole’s current
architecture while preserving crosschain functionality.

Supporting both onchain and offchain quotes ensures compatibility across
different chain environments without imposing a single quoting model.

The proposal is a targeted infrastructure upgrade and does not introduce changes
to xWELL’s economic role or governance design.

## Voting Options

- **For** — Approve the upgrade to the Wormhole executor model with dual quote
  support.
- **Against** — Do not approve the proposal.
- **Abstain** — Neither support nor oppose.
